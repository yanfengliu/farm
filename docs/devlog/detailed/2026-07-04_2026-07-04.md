# Detailed Devlog: 2026-07-04 to 2026-07-04

## 2026-07-04

- Action: Started the game from an empty seed repo, gathered product requirements, and established the initial design and architecture documentation.
- Result: The canonical design describes a desktop-first browser idle farm where tiny autonomous workers handle planting, watering, harvesting, and hauling on a top-down pixel grid.
- Reasoning: `civ-engine` is well-suited to deterministic worker logistics and testable simulation. Phaser is a practical renderer for pixel-art tiles, sprites, camera, input, and audio.
- Notes: The first implementation slice should prove the full carrot-style loop before expanding content.

- Action: Implemented the first playable game slice.
- Result: The app now runs in Vite with a Phaser farm canvas, DOM HUD, toolbar, inventory/goals/crop-mix/inspect panels, visible hotkeys, pause and speed controls, localStorage autosave, simple audio cues, and browser debug APIs. The headless simulation covers planting, watering, harvesting, storage delivery, seed purchasing, manual selling, crop overflow behavior, land purchase, build painting, undo/redo, deterministic replay, and text rendering.
- Reasoning: A small but complete loop proves the source-of-truth simulation, renderer boundary, and UI command bridge before adding more crops, content, or polish.
- Validation: `npm test`, `npm run typecheck`, `npm run lint`, `npm run build`, Playwright screenshots, and a browser debug API smoke check were used during the implementation loop.
- Notes: Vite build initially failed because the root `civ-engine` export barrel pulled in Node-only replay/file modules. The fix was a narrow browser-safe adapter at `src/game/simulation/civEngine.ts`.

- Action: Ran an in-process adversarial review after the first playable slice.
- Result: Fixed the Phaser real-time advancement path so fractional frame deltas accumulate into simulation ticks, made storage capacity recalculate whenever storage is placed or removed, auto-sold over-capacity crops after storage removal, and added a collapsible right side panel.
- Reasoning: The headless debug API can advance large time chunks, so browser real-time needed a separate check. Storage capacity is a map-derived rule and must not drift when the player bulldozes or overwrites bins.
- Validation: Added a focused storage-capacity regression test. Browser verification should include both debug time advancement and passive real-time tick progression.
- Notes: External reviewer tooling was unavailable in this session, so this review was completed in-process.

- Action: Closed the remaining MVP mechanic gap for global tool upgrades.
- Result: Added worker boots and watering can upgrades, both purchased with coins from the Goals panel and saved in deterministic farm state.
- Reasoning: The product decisions called for tools to be global upgrades. A small upgrade pair gives players a meaningful coin sink without adding physical tool inventory or a larger crafting economy.
- Validation: Added a simulation contract for upgrade purchases and debug text projection.

- Action: Added the full tier path to the player-facing Goals panel.
- Result: Tier labels, milestones, crop unlocks, and reward summaries now come from shared tier content and render as a compact progression list.
- Reasoning: The HUD should show the single next milestone, while the side panel should expose the broader tier path without requiring a separate screen.

- Action: Ran an LLM playtest and polish loop focused on the MVP player feel.
- Result: Reduced persistent chrome height, made toolbar and panel buttons smaller, added themed range inputs, improved pixel-art rendering for grass, paths, plots, wells, storage, crops, and workers, added deterministic worker sprite offsets for overlapping workers, and moved tool upgrades above the tier reference list in the Goals panel.
- Reasoning: The first playable build worked mechanically but read too much like a UI shell with plain square art. The revised pass gives the farm more identity, keeps controls compact, and makes early coin-spend decisions easier to find.
- Validation: Captured before/after desktop and smaller desktop screenshots, tested progressed farm state, verified compact toolbar interaction, panel collapse, Crop Mix, Inventory, and Goals surfaces with Playwright.

- Action: Reworked pixel rendering toward object-level detail instead of abstract shapes.
- Result: Locked land now has dark weeds and stones, owned grass has tufts/flowers/rocks, paths have stone and highlight detail, plots have borders/furrows/seeds, wells have stone rings/water/roof/crank/bucket pixels, storage has planks/latches/sacks, crops draw as crop-specific rows, and workers have hats, faces, clothes, arms, boots, cargo, and task tools.
- Reasoning: The prior polish pass improved the screen, but many objects still read as colored rectangles. The new pass uses small recognizable pixel clusters inside the existing 32px tiles so the farm stays readable without needing imported sprite sheets.
- Validation: Captured starter, crop-stage, and progressed Playwright screenshots with zero page errors.
