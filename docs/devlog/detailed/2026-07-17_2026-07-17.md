# 2026-07-17

## Living farmhands and storybook chrome

- Brainstormed per the open creative mandate and specced before building (`docs/superpowers/specs/2026-07-17-living-farmhands-storybook-chrome-design.md`, committed separately). Rejected a global dawn/dusk light cycle (intersects every palette-sampling contract across the browser suites), new simulation wildlife (save/replay surface, not a graphics pass), and building/crop portraits (deferred until tile art shares the painter type). Shipped: named farmhands, Inspect pixel portraits, presentation-only work celebrations, and layout-neutral chrome accents.
- The enabling discovery: every view art module's Phaser import is type-only, and the draw functions only ever call `fillStyle` and `fillRect`. A two-method `PixelPainter` interface names that contract; `drawFarmhand` retyped against it with zero behavior change (Phaser Graphics satisfies it structurally), and `drawFarmhandPortrait` renders the standing pose from a synthetic frozen state kept local to the art module. `src/ui/inspectPortrait.ts` adapts a DOM canvas 2D context in ~20 lines, so the Inspect portrait is the scene's own sprite at 3x — never a divergent copy.
- Names are content-only projection (`src/game/content/farmhands.ts`: Fern, Alder, Poppy, Rowan by worker id; unknown ids stay readable as "Farmhand N"). Nothing enters `FarmState` or the save.
- Work celebrations: `farmWorkEffects.ts` spawns from lifetime-stat diffs between snapshots at each worker whose task matches the grown verb — a pure, unit-tested decision — then animates dust/droplets/sparkles on presentation time in the effects lane, capped at 48 live effects. Paused farms spawn nothing new (stats frozen) while in-flight pixels finish, matching the documented ambience rule.
- Chrome: stepped harvest-gold heading underline (backgrounds only), HUD chip pixel frames (inset shadows only), and a one-shot Coins pulse replayed naturally because any coin change rewrites the HUD chip element. No element grows; panel overflow measured 0 at both supported sizes with the portrait card open.
- Verification: unit tests pin profile completeness, portrait purity/bounds/determinism/per-hand distinctness, spawn decisions, draw-path pixels near the working tile, and lifetime expiry. Browser contracts pin the inspected name + painted portrait (>80 opaque pixels, >5 distinct colors, `image-rendering: pixelated`), the coin pulse, and effect purity — a paused farm's `render_game_to_text()` and save bytes are byte-identical across a second of live effect frames. Fresh screenshots at 1280x800 and 1024x720 show the Fern card fitted with zero overflow; a bounded 24-step visual loop against the live server ran with zero console/page errors.

## The 15-second browser budget convention died at 56 files

- The enrichment's gate run surfaced rotating 15s timeouts across browser suites — different files each run, all passing in isolation. Lowering the fork cap to 4 did not help, which killed the pure-load theory: the suite's slowest tests measure 10-12 seconds of genuine solo work (cold per-file Vite transform plus canvas pixel scans), and with 22 Chromium-launching files sharing 6 forks, long suites pin forks and stack the tail past any 15s budget.
- Causality check before touching anything: stashing the enrichment and timing the worst offender on a clean checkout measured 11.8s solo on `main` against 9.8s with the enrichment applied — the feature work made nothing slower; suite growth consumed the headroom that had been hiding a latent calibration problem, exactly as the 2026-07-15 fork-cap work predicted could recur.
- Fix: one mechanical sweep of every inline `}, 15000)` in `tests/browser/` to `}, 30000)` (22 raised in the sweep, plus the five chronic files raised while diagnosing). Budgets now encode real cost plus scheduling reality; hangs still fail. Full suite green 56/56 three consecutive runs, with typecheck, lint, and build clean.
