## Recursive-loop alignment: replayable bundles, honest verification, append-only history

**Timestamp:** 2026-07-08 (evening).

**Action.** Aligned the visual loop with the cross-game recursive-improvement contract (civ-engine 1.6.x). Game side: a dev-only `SessionRecorder` attaches to the live world (re-attached on reset) and `window.__farmDebug.exportBundle()` disconnects it for a terminal snapshot, exports the replayable `SessionBundle`, and re-attaches a fresh recorder; `tsconfig` gains `vite/client` types for the DEV gate. Loop side: `scripts/llm-visual-loop.mjs` writes `latest.bundle.json`, verifies it in-process with `SessionReplayer.selfCheck` against the real `createFarmGame` world factory seeded from the recorded snapshot (skipped honestly on external `FARM_PLAYTEST_URL` runs), archives the previous run to `output/playwright/llm-visual-loop-history/<stamp>/` instead of `rm -rf`, and appends each run manifest to `history/ledger.jsonl`. Report side: findings author as `unverified`; deterministic (artifact-computed) findings flip to `verified` + `verificationMethod: 'metric'` + a bundle evidence ref ONLY under strong replay verification (ok, >0 checked segments, 0 skipped); LLM/engine-derived findings never auto-verify; `makeFinding` stamps `minimalImprovementFindingSchemaVersion` and strict-asserts verified findings; run manifests carry `sessionId`/`bundleId` so ledger rows join to archived bundles. `scripts/llm-playtest-replay.mjs` defaults to the live bundle and reports `selfCheckStrongOk`.

**Adversarial review.** Codex CLI (gpt-5.5 xhigh) + one in-process subagent, both grounded in the live code with end-to-end probes. Codex: the verify world factory's hard-coded seed diverged self-check on `config.seed` for every real bundle (probed; fixed by seeding from `snapshot.config.seed`), and live `toBundle()` yielded vacuous zero-segment checks on short runs (fixed with the disconnect-export-reattach pattern). In-process HIGH: `SelfCheckResult.skippedSegments` is an ARRAY and three sites compared it to the number 0, so the verified-by-metric upgrade could never fire while all 57 tests passed against number-shaped fixtures — fixed with boundary normalization + a tolerant `strongReplayVerification` + fixtures exercising the real array shape (plus a partially-skipped case). Also fixed from review: manifest join fields. Accepted notes: crashed runs break the one-back comparison chain (history fallback is a follow-up); Windows `fs.rename` on held-open files aborts at startup like the old `rm` did. This work also surfaced and rode on the engine 1.6.1 patch (browser-safe session ids — `node:crypto` import took the page bundle down).

**Validation.** TDD red-first on the contract updates (5 failing), then implementation; the engine bump's pre-existing schema-pin failure resolved via minimal stamping rather than pin edits. Full gates green: `npm test` (129 passed, 13 files), `npm run typecheck`, `npm run lint`, `npm run build`. Docs updated: `docs/testing-playtest.md` (bundle artifact, append-only history, honest verification wording, replay default path).

## playtest:recursive: one-command proposal-only pass

**Timestamp:** 2026-07-08 (late evening).

**Action.** Added the fleet-uniform pass entry point: `npm run playtest:recursive` runs the visual loop, selects the highest-severity open fix-classified finding (`nextAction` autoFix/manualFix, non-rejected) from the canonical `latest.json` packet, and writes `latest.pass-manifest.json` plus a row in `output/playwright/llm-visual-loop-history/passes.jsonl` using the fleet outcome vocabulary (`no-fix-candidate | proposal-only | run-failed`) via the engine's `createImprovementRunManifest`. Farm stays proposal-only by design — the driving agent is the fix arm (fix, rerun, compare the packet's `comparison`). Pure decisions live in `scripts/llm-visual-loop/recursive-pass.mjs` with unit tests; the script is a thin spawner. Also hardened `archivePreviousRun` against Windows EPERM directory locks (brief retry, then per-child move fallback, then warn-and-continue) after the first real archive hit a held directory handle.

**Verification.** TDD on the pure module (selector ranking/skips, manifest validation via engine `assertImprovementRunManifest`, outcome vocabulary); live smoke: a 6-step pass produced outcome `proposal-only` with candidate `visual-loop-ended-with-guidance` and honestly reported the short run's vacuous self-check as weak (findings stayed unverified). Gates green: `npm test`, `npm run typecheck`, `npm run lint`, `npm run build`. Docs: `docs/testing-playtest.md` pass section.
