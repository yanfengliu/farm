# 2026-07-16

## Deflaked the two intermittent browser suites

- Result: `speedPreference` "Space activates a focused button without also pausing the farm" and `annotations` "pins follow camera movement and View restores the captured composition" both failed intermittently — under full-suite load and, for the annotations test, occasionally in isolation. Both are now green five consecutive runs of the failing pair and two consecutive full-suite runs. No app code changed for either: both were test races against the same architectural fact, that every panel and HUD DOM update rides the next Phaser `ui.render()` frame.
- speedPreference was a genuine test-logic bug, not a tight budget. After pressing Space on the Goals tab it waited on the bare selector `#panel-content h2` — which is already satisfied by the still-rendered previous panel — then read the heading before the frame swapped Goals in, producing the recorded `expected 'Inventory' to match /Tier/`. The wait is now on the Goals heading text itself, and the speed read moved after that wait so a wrongly-paused farm would actually be visible to the assertion.
- The annotations failure took three instrumented probes to pin down, and the first two theories were wrong. It was not frame-stall budget exhaustion (raising the 2s waits to 5s made the failure *reliable* rather than fixing it, by letting the test reach the real blocker), and it was not per-frame panel churn (a MutationObserver recorded zero mutations across three seconds of an open draft). The trace that settled it: after saving note #1, the stale draft card is legally still in the panel for one frame because `saveDraft` refreshes the overlay synchronously while the panel rebuild waits for the next `ui.render()`. Around the second draft's open, the panel therefore transitions stale-save-card → aim-card → draft-card, and each rebuild detaches the Cancel node, so `page.click('[data-command="cancel-annotation"]')` can keep resolving nodes that are replaced beneath it — Playwright's `element was detached from the DOM, retrying` loop. A real player only ever sees the settled states.
- Fix: the cleanup clicks moved off the panel DOM entirely. `Escape` is the documented player path — the first returns the open draft to aiming, the second exits aiming — and the keyboard state machine is synchronous, so it cannot race a rebuild. The draft opening remains the actual assertion (the existing pin did not intercept the aim click), and the aria-pressed poll still verifies the exit state. The three tight 2000ms waits stay at 5000ms, sized for parallel-suite frame stalls; the test budget rose 30s → 45s.
- Evidence trail: `render_game_to_text()` state tracing showed the exact sequence healthy and stable for 2+ seconds (`aiming=false draft=true`, Cancel present and visible, textarea focused), which exonerated the app state machine and isolated the failure to click-time node identity.

## Opened the legacy-save browser boot lane

- Gap (named in yesterday's summary): every browser test that seeds a save uses a current-schema snapshot, so `normalizeFarmState`'s additive migrations had never run in a browser — only against the unit suite's in-memory storage double. A migrated state can be unit-valid and still throw in the renderer, and nothing would have caught that class.
- `tests/browser/legacySaveMigration.test.mjs` seeds a genuinely old save built by stripping every validation-optional field a pre-Village-Harvest, pre-ecology farm lacks — `wildlife`, `community`, `upgrades`, both optional lifetime counters, and every `pumpkin` crop-record entry, including inside a stripped undo snapshot — then boots it through the real path: `main.ts`, Phaser, autosave. It asserts the save loads rather than being refused as unreadable, the played tick survives, every migration lands (Pip and Mallow with four fish, zeroed upgrades, community state, pumpkin entries), the migrated ecology reaches `render_game_to_text()`, the canvas actually paints, zero page errors, and — closing the round trip — autosave writes back a current-schema save so the next boot is no longer a legacy boot.
- Mutation proof: commenting out `state.wildlife ??= createInitialWildlifeState()` in `normalizeFarmState` fails the contract with `TypeError: Cannot read properties of undefined (reading 'ducks')` — precisely the unit-valid-but-breaks-downstream failure class the lane exists to catch. Restoring the line returns it to green.
- Verification: full gates pass 49 of 49 files twice consecutively, with typecheck and lint clean and the build carrying only the established Phaser chunk advisory.
